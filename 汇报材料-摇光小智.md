## 一、产品定位与概述

摇光小智是一个**基于需求模型的系统工程协作者**，支持**需求查询、追溯分析、质量审查、需求生成与工程过程引导**五大核心能力。

### 1.1 产品背景

在MBSE（模型驱动系统工程）项目中，需求管理是系统工程的起点与核心。传统需求分析与管理面临以下挑战：

| 挑战类型 | 具体表现 |
|----------|----------|
| **知识获取难** | 需求分析依赖资深工程师经验，知识传承困难 |
| **追溯分析难** | 需求与设计、测试之间的追溯链路复杂，人工分析效率低 |
| **质量审查慢** | 军工、航空航天行业需求合规要求严格，人工审核易遗漏 |
| **需求编写慢** | 规范化需求编写门槛高，新手工程师产出质量不稳定 |
| **过程指导缺** | 系统工程过程复杂，工程师难以掌握完整流程 |

摇光小智正是为解决上述痛点而生，将AI能力深度嵌入需求管理全流程，成为工程师的智能协作伙伴。

### 1.2 目标用户

| 用户角色 | 使用场景 |
|----------|----------|
| **需求分析师** | 需求编写、规范查询、历史需求检索、需求生成辅助 |
| **系统工程师** | 追溯分析、质量审查、跨文档关联分析、过程引导 |
| **项目经理** | 了解项目需求状态、获取需求质量报告、过程监控 |

### 2.1 模型感知型查询（Model-aware Query）

> **定位**：让AI真正"理解"项目，而非仅仅检索关键词

#### 核心能力

| 能力维度 | 说明 |
|----------|------|
| **需求模型实时感知** | 通过MCP协议实时读取当前项目的需求模型结构、元素属性与关联关系 |
| **上下文智能理解** | AI基于当前打开的模型视图与选中元素，自动关联相关需求上下文 |
| **多源知识融合** | 整合项目数据、知识库、行业规范三大信息源，提供精准回答 |
| **自然语言交互** | 用户以日常语言提问，系统自动转化为模型查询并返回结构化结果 |

#### 典型应用场景

- *"这个功能模块有哪些子需求？"*
- *"与安全相关的需求有哪些？按优先级列出"*
- *"导弹控制系统的性能指标需求是什么？"*

#### 解决的核心问题

| 行业痛点 | 摇光解决方案 | 商业价值 |
|----------|--------------|----------|
| 大型项目需求量达数千条，人工检索耗时 | AI秒级定位目标需求，支持语义模糊匹配 | **检索效率提升20倍**，单次查询从10分钟降至30秒 |
| 传统搜索仅支持关键词匹配，易遗漏 | 模型语义理解+知识图谱关联检索 | **召回率提升40%**，减少信息遗漏风险 |
| 新员工不熟悉项目结构，上手慢 | AI助手随时答疑，自动关联上下文 | **新人培养周期缩短50%** |

### 2.2 需求追溯与影响分析（Traceability & Impact）

> **定位**：一键洞察变更影响，让风险"无处遁形"

#### 核心能力

| 能力维度 | 说明 |
|----------|------|
| **全链路追溯** | 自动构建需求-设计-测试-交付的完整追溯链路，支持正向/逆向追溯 |
| **变更影响分析** | 当某条需求变更时，自动识别所有受影响的下游元素（设计、接口、测试用例） |
| **覆盖率可视化** | 实时展示需求到测试的覆盖情况，识别未覆盖或过度覆盖的需求 |
| **孤立需求检测** | 自动发现无上游来源或无下游分解的"孤立需求"，防止需求遗漏 |

#### 典型应用场景

- *"如果我修改这条接口需求，会影响哪些测试用例？"*
- *"这条系统需求是从哪个用户需求派生的？"*
- *"哪些需求还没有对应的测试用例？"*

#### 解决的核心问题

| 行业痛点 | 摇光解决方案 | 商业价值 |
|----------|--------------|----------|
| 需求变更影响评估依赖人工，易遗漏导致返工 | AI自动分析变更波及范围，生成影响报告 | **变更评估效率提升10倍**，返工成本降低60% |
| 追溯链路人工维护困难，数据不一致 | 基于模型自动构建追溯关系，实时同步 | **追溯数据准确率≥98%** |
| 军工项目合规审查要求追溯完整性 | 一键生成追溯矩阵报告，支持审查交付 | **审查准备时间从3天降至2小时** |

### 2.3 需求质量分析与审查（Requirement Quality Review）

> **定位**：AI驱动的质量门禁，将问题拦截在源头

#### 核心能力

| 能力维度 | 说明 |
|----------|------|
| **多维度质量评估** | 覆盖完整性、一致性、可追溯性、可验证性、符合性五大维度 |
| **规范符合性校验** | 内置GJB、航空航天等行业标准规则库，支持自定义规则扩展 |
| **智能问题定位** | 精确标注问题位置，并提供具体修改建议 |
| **批量审查能力** | 支持整库/整模块批量扫描，快速获取质量全景 |

#### 质量评估维度

| 维度 | 评估内容 | 典型问题示例 |
|------|----------|--------------|
| **完整性** | 必填字段是否齐全，描述是否充分 | "该需求缺少验证方法说明" |
| **一致性** | 需求之间是否存在逻辑矛盾 | "需求A要求响应时间<1s，需求B要求<2s，存在冲突" |
| **可追溯性** | 是否具备完整的追溯链 | "该需求无上游来源，疑似孤立需求" |
| **可验证性** | 是否可被测试或验证 | "需求描述过于模糊，无法制定测试方案" |
| **符合性** | 是否符合行业规范要求 | "需求编号格式不符合GJB要求" |

#### 解决的核心问题

| 行业痛点 | 摇光解决方案 | 商业价值 |
|----------|--------------|----------|
| 人工评审效率低，200条需求需2-3天 | AI批量扫描，200条需求15分钟内完成 | **评审效率提升10倍以上** |
| 评审标准不统一，结果因人而异 | 统一规则引擎，确保评审一致性 | **质量标准化程度提升** |
| 问题发现滞后，下游返工成本高 | 需求阶段前置质量门禁 | **下游返工成本降低70%** |

#### 核心指标

| 指标 | 目标值 | 业务意义 |
|------|--------|----------|
| 问题检出准确率 | **≥85%** | 确保发现的问题真实有效 |
| 误报率 | **<5%** | 避免无效工作量 |
| 单次审查响应时间 | **≤3秒** | 支持实时交互体验 |

## 三、交互界面设计

### 3.1 对话系统界面

摇光小智采用类似Cursor、Copilot的嵌入式对话框设计，在需求管理系统右侧嵌入对话界面。

**界面特点**：

- **气泡式界面**：区分"用户"（右侧）与"AI助手"（左侧）的消息气泡
- **流式响应**：AI回复时实时逐字显示，内容即时上屏
- **富文本支持**：完美支持代码高亮、表格、公式等格式
- **调试面板**：可折叠的调试区域，包含错误信息、元数据、性能指标

**交互操作**：

| 操作 | 说明 |
|------|------|
| **发送** | 点击发送按钮或按快捷键提交问题 |
| **停止** | 生成过程中可随时中断 |
| **清空** | 清空当前会话记录，开始新对话 |
| **调试** | 切换调试面板的展开/收起状态 |

### 3.2 合规校验界面

**校验结果展示**：

- **问题列表**：按严重程度排序展示所有发现的问题
- **定位功能**：点击问题可自动跳转至需求文档对应位置
- **改进建议**：每个问题附带具体的修改建议
- **统计概览**：展示问题总数、各类型问题分布
