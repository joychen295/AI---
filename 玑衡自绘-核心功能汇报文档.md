# 玑衡自绘 · 核心功能汇报

---

## 一、产品定义

**玑衡自绘**是一个集成于玉衡建模工具的AI辅助建模系统，实现自然语言到SysML模型的智能转换。

---

## 二、核心功能

### 功能一：自然语言建模

| 维度               | 内容                                                                                                    |
| ------------------ | ------------------------------------------------------------------------------------------------------- |
| **功能名称** | 自然语言建模                                                                                            |
| **功能描述** | 用户通过自然语言描述建模需求，系统自动生成符合SysML规范的模型结构。                                     |
| **应用场景** | ① 快速建模：系统架构师快速将设计思路转化为模型结构                                                     |
| **价值量化** | ・建模效率提升10倍 `<br>`・建模周期从2-4周缩短至2-3天 `<br>`・降低SysML建模门槛，普通工程师也能建模 |

---

### 功能二：XMI格式导出与集成

| 维度               | 内容                                                                                                                                                                                                                                       |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **功能名称** | XMI格式导出与集成                                                                                                                                                                                                                          |
| **功能描述** | AI生成的模型自动转换为XMI标准格式，支持与天玑/玉衡建模工具无缝对接。处理流程包括：语义理解与意图识别→模型元素生成/修改→XMI格式转换与校验→导入建模工具。系统内置XMI格式解析、生成、校验能力，解决XMI文件冗余度高、嵌套层次深的技术难题。 |
| **应用场景** | ① 模型交换：AI生成结果导入现有建模工具继续编辑`<br>`② 模型复用：历史模型作为模板导出后二次利用`<br>`③ 工具协作：与其他支持XMI格式的建模工具互通                                                                                     |
| **价值量化** | ・与现有工具无缝集成，无需额外学习成本`<br>`・支持SysML v1.6标准，兼容主流建模工具`<br>`・工程化XMI处理能力，解析生成校验一体化                                                                                                        |

---

### 功能三：MBSE知识问答

| 维度               | 内容                                                                                                                                                                                                                                                                                          |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **功能名称** | MBSE知识问答                                                                                                                                                                                                                                                                                  |
| **功能描述** | 用户可随时询问MBSE相关问题，系统基于四层知识库提供专业解答：本体论知识（SysML规范、MBSE理论基础）、方法论知识（公司内部建模方法论、最佳实践）、案例库（历史项目典型模型）、外部知识库（OMG官方文档、学术论文）。支持"SysML中Block和Class有什么区别？""有没有类似的控制系统参考模型？"等问题。 |
| **应用场景** | ① 规范查询：快速了解SysML规范细节与使用方法`<br>`② 方法论学习：查询公司内部建模方法与最佳实践`<br>`③ 参考模型：检索历史项目中相似的典型模型作为参考                                                                                                                                    |
| **价值量化** | ・新员工培养周期缩短50%以上`<br>`・工程师无需反复查阅手册，学习成本显著降低`<br>`・知识查询即时响应，提升工作效率                                                                                                                                                                         |

---

### 功能四：模型代码补全

| 维度               | 内容                                                                                                                                                                                                                         |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **功能名称** | 模型代码补全                                                                                                                                                                                                                 |
| **功能描述** | 在SysML v2.0文本化建模过程中，系统自动预测并提示补全内容。用户输入"block Motor"，系统自动建议补全属性、端口定义；根据上下文预测下一步可能的建模操作。类似GitHub Copilot的代码补全能力，针对KerML文本化建模语言进行专项优化。 |
| **应用场景** | ① 快速输入：模型代码编写过程中自动补全减少输入量`<br>`② 语法提示：避免语法错误，提升模型规范性`<br>`③ 上下文预测：根据当前模型结构智能预测下一步操作                                                                  |
| **价值量化** | ・建模输入效率大幅提升`<br>`・减少语法错误，提高模型质量`<br>`・充分发挥SysML v2.0文本化建模优势                                                                                                                         |

---

### 功能五：智能建议与优化

| 维度               | 内容                                                                                                                                                                                                                                                                          |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **功能名称** | 智能建议与优化                                                                                                                                                                                                                                                                |
| **功能描述** | 系统分析当前模型结构，主动提供优化建议。建议类型包括：遗漏提示（提示可能遗漏的必要元素）、关系优化（发现不合理的关联关系并建议修正）、规范符合性（检测不符合规范的建模方式）、最佳实践（推荐更好的建模模式）。所有建议附带推理依据，如"根据需求文档第3.2节，该接口是必需的"。 |
| **应用场景** | ① 质量审查：模型完成后自动检测潜在问题与遗漏`<br>`② 规范校验：确保模型符合SysML规范与行业标准`<br>`③ 优化指导：获取建模最佳实践建议，持续改进模型质量                                                                                                                  |
| **价值量化** | ・提升模型规范性与完整性`<br>`・降低人工审查工作量`<br>`・建议可解释，支持工程师自主判断                                                                                                                                                                                  |

---

### 功能六：人机协同建模

| 维度               | 内容                                                                                                                                                                                                                                   |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **功能名称** | 人机协同建模（Human-in-the-Loop）                                                                                                                                                                                                      |
| **功能描述** | 采用"AI辅助、人工确认"的设计理念。AI生成模型草稿后，工程师审核、修改、确认后才写入正式模型库。支持部分接受（工程师可选择接受部分生成结果，拒绝或修改其他部分）。根据项目风险等级调整AI自主权限，提供保守/标准/激进三种自动化级别设置。 |
| **应用场景** | ① 关键系统建模：AI提供建议，工程师最终确认关键决策`<br>`② 增量修改：选择性接受AI建议，保持对模型的控制权`<br>`③ 风险管控：高风险项目采用保守模式，确保人工把关                                                                  |
| **价值量化** | ・确保工程师对最终模型的控制权`<br>`・AI辅助与人工判断相结合，质量与效率兼顾`<br>`・满足军工、航空航天等高安全要求行业需求                                                                                                         |
