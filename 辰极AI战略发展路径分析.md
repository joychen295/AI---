## 一、背景与问题

### 1.1 项目立项的表面理由

辰极智脑AI中台项目的立项，有一系列显而易见的理由：

- AI是当前最大的技术风口
- 行业内主流厂商都在布局大模型与AI能力
- 客户开始询问和需要AI相关的解决方案
- 公司需要抓住这一轮技术变革的机遇

这些理由都是真实的，也是推动项目立项的直接动因。

### 1.2 需要回答的核心问题

然而，上述理由解释的是辰极智脑**出现的历史必然性**——为什么这个时间点要做AI。但它们并没有回答更关键的商业问题：

| 表面理由         | 未回答的问题                               |
| ---------------- | ------------------------------------------ |
| AI很火           | 火的东西很多，为什么一定是我们做？         |
| 大家都在做大模型 | 我们做大模型有什么优势？                   |
| 客户需要AI       | 客户可以选择任何AI供应商，为什么选择我们？ |
| 公司需要抓住风口 | 抓住风口之后，怎么变成持续的收入？         |

**"因为AI是风口所以做"不是商业模式，也不是护城河。**

这就引出了本汇报需要回答的核心问题：

> **1. 辰极智脑中台的商业模式是什么？如何实现变现？**
>
> **2. 真正的护城河是什么？凭什么别人做不了、追不上？**
>
> **3. 公司AI团队的发展路径应该怎么走？**

### 1.3 对护城河的重新审视

**什么不是护城河？**

| 常见误解   | 为什么不是护城河                                                    |
| ---------- | ------------------------------------------------------------------- |
| 大模型能力 | DeepSeek、Kimi、GLM等大模型是公开基础设施，我们能用，竞争对手也能用 |
| AI技术团队 | 技术人员可以流动，技术方案可以复制                                  |
| 先发优势   | 没有壁垒的先发优势很快会被追平                                      |

**什么才是真正的护城河？**

AI中台真正的护城河，不是模型本身，而是**不可复制的知识资产与场景绑定**。这种护城河分为三个层级，且层级之间存在递进关系：

#### 护城河层级一：知识资产沉淀

这是构建护城河的基础层，包含三类知识资产：

| 知识类型             | 具体内容                             | 当前状态       |
| -------------------- | ------------------------------------ | -------------- |
| **本体论知识** | SysML规范、MBSE理论基础、建模元语言  | 已沉淀于知识库 |
| **方法论知识** | 公司内部建模方法论、项目交付最佳实践 | /              |
| **案例库知识** | 历史项目典型模型、行业标准模板       | 持续积累中     |

这三类知识资产支撑了中台的三大核心能力：

| 核心能力                 | 依赖的知识资产             | 功能体现                          |
| ------------------------ | -------------------------- | --------------------------------- |
| **智能知识问答**   | 本体论+方法论+案例库       | 基于知识库的专业问答，准确率≥85% |
| **智能代码补全**   | 本体论（SysML语法）+案例库 | 基于KerML文本的上下文感知补全     |
| **主动误检与建议** | 方法论+行业规范            | 模型结构分析、问题检测、优化建议  |

**关键认知**：知识资产沉淀是护城河的起点，但这一层本身是可以被复制的——竞争对手如果投入足够时间，也可以建立类似的知识库。真正不可复制的是下一层。

#### 护城河层级二：产品落地与场景绑定

知识资产只有通过产品落地到客户场景中，才能转化为真正的护城河。这一层的关键是**让AI能力嵌入客户的日常工作流程**：

| 产品                 | 落地场景              | 形成的绑定                     |
| -------------------- | --------------------- | ------------------------------ |
| **摇光小智**   | 嵌入瑶光需求管理系统  | AI成为需求分析师的日常工作伙伴 |
| **玑衡自绘**   | 嵌入天玑/玉衡建模工具 | AI成为建模工程师的效率工具     |
| **Create平台** | 独立的工业AI智能平台  | AI成为系统工程完整解决方案     |

**关键认知**：摇光小智和玑衡自绘的战略价值，不仅在于"增值收费"，更在于它们是**知识资产进入客户场景的载体**。没有产品落地，知识资产只是内部积累；有了产品落地，知识资产才能触达客户、产生价值、并进入下一层护城河的构建。

#### 护城河层级三：客户数据沉淀与飞轮效应

当产品进入客户场景后，客户的使用行为会产生数据，这些数据会进一步强化AI能力：

| 数据类型               | 来源                           | 价值                     |
| ---------------------- | ------------------------------ | ------------------------ |
| **客户项目知识** | 客户在系统中的需求、模型、文档 | 形成客户专属的知识图谱   |
| **使用行为数据** | 客户的查询、操作、选择         | 优化AI推荐与补全的精准度 |
| **反馈修正数据** | 客户对AI输出的接受/拒绝/修改   | 持续迭代模型，提升准确率 |

这形成了一个**数据飞轮**：

```
客户使用产品 → 产生使用数据 → 沉淀为客户专属知识图谱 → AI更懂客户项目
      ↑                                               │
      │                                               ▼
      └────── 客户更依赖 ← AI回答更精准、建议更有用 ←───────┘
```

**关键认知**：这一层是真正不可复制的护城河。竞争对手可以复制技术方案，但无法获取我们在客户环境中积累的真实数据。客户用得越久，知识图谱越丰富，AI越懂客户的项目上下文，切换成本越高。

#### 护城河的形成路径

```
┌──────────────────────────────────────────────────────────────────────────┐
│                       护城河构建路径                                      │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  第一层：知识资产沉淀（可复制，但需要时间）                                 │
│  ──────────────────                                                      │
│  本体论知识 + 方法论知识 + 案例库知识                                      │
│       │                                                                  │
│       ▼                                                                  │
│  第二层：产品落地与场景绑定（中等壁垒）                                    │
│  ──────────────────────────                                              │
│  摇光小智（需求场景）+ 玑衡自绘（建模场景）+ Create（完整平台）             │
│       │                                                                  │
│       ▼                                                                  │
│  第三层：客户数据沉淀与飞轮效应（不可复制）                                │
│  ────────────────────────────                                            │
│  客户项目知识 → 专属知识图谱 → AI越用越懂 → 高切换成本                    │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

### 1.4 商业闭环的逻辑

传统软件的商业模式是"一次性交付"：卖出License，交付完成，价值兑现结束。

辰极智脑AI中台的变现逻辑不同——**不是一次性卖产品，而是通过持续服务持续收费**。

核心变现路径是：

> **客户使用产品 → 积累知识资产 → AI越来越懂客户 → 客户持续付费**

最终产生三种商业收益：

- **续费收入**：客户因为积累了太多数据和知识，不愿意切换
- **增购扩展**：从单个产品扩展到全流程解决方案
- **转介绍收入**：满意客户带来新客户

这一逻辑体现为四个阶段的商业闲环：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      商业闭环四阶段                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│ 阶段一：试用验证                                                         │
│ ──────────────                                                          │
│ • 入口：捆绑现有软件试用 / 试点项目                                       │
│ • 目标：让客户体验AI能力，验证实际价值                                    │
│ • 关键动作：快速展示效率提升（如检索效率20倍、评审效率10倍）               │
│                     │                                                   │
│                     ▼                                                   │
│ 阶段二：深度使用                                                         │
│ ──────────────                                                          │
│ • 入口：试用转付费                                                       │
│ • 目标：AI解决客户的实际工作问题，成为日常工具                            │
│ • 关键动作：培训用户、建立使用习惯、开始积累客户数据                       │
│ • 变现：首次付费（5-10万/年）                                            │
│                     │                                                   │
│                     ▼                                                   │
│ 阶段三：全流程嵌入                                                       │
│ ──────────────────                                                      │
│ • 入口：单点使用扩展为全流程                                              │
│ • 目标：AI覆盖需求-设计-验证完整链路                                      │
│ • 关键动作：积累客户项目知识、构建客户专属知识图谱、形成"越用越懂"效应    │
│ • 变现：增购扩展（10-50万/年）                                          │
│                     │                                                   │
│                     ▼                                                   │
│ 阶段四：高价值绑定                                                       │
│ ──────────────────                                                      │
│ • 状态：客户的知识资产已沉淀在系统中                                      │
│ • 效果：切换到竞品意味着丢失积累的知识图谱、重新培训团队、重建工作流程     │
│ • 变现：持续续费 + 转介绍新客户                                          │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**总结**：客户不是被"锁定"，而是因为在系统中积累了太多有价值的知识资产，切换成本太高，所以选择持续合作。这种模式下，**客户用得越久，我们赚得越多**。

基于以上分析，下文将详细阐述护城河如何通过具体产品落地，以及商业模式的实施路径。

## 二、产品落地与商业拓展

基于辰极智脑中台，我们规划了三个具体的**产品落地切入点**（摇光小智、玑衡自绘、Create平台）。本章将详细汇报这三个产品**"到底做什么"**、**"我们将怎么做"**，并基于此拓展出更广阔的**商业与应用可能性**。

### 2.1 摇光小智：从"嵌入式助手"到"企业级知识伴侣"

**【我们要做什么：具象产品能力】**

摇光小智是嵌入在**瑶光需求管理系统**中的智能协作者，核心目标是解决需求分析师"查不到、理不清、写不准"的痛点。我们将重点实现五大核心能力：

1. **模型感知型查询**：不只是搜关键词，而是通过MCP协议实时读取项目上下文，让AI真正"理解"项目。

   - *典型场景*："这个功能模块有哪些子需求？""与安全相关的需求有哪些？""导弹控制系统的性能指标需求是什么？"
   - *价值*：解决大型项目需求检索耗时、关键词搜索易遗漏的痛点，**显著提升检索效率**，缩短新人上手周期。
2. **需求质量审查**：基于军工/行业规范的自动化审查，覆盖完整性、一致性、可追溯性、可验证性、符合性五大维度。

   - *典型场景*：自动扫描需求文档，指出"该需求缺少验证方法说明""需求A与需求B存在响应时间冲突""该需求无上游来源，疑似孤立需求"。
   - *价值*：解决人工评审周期长、评审标准不统一的痛点，**大幅提升评审效率**，降低下游返工成本。
3. **变更影响分析**：一键洞察变更波及范围，自动构建需求-设计-测试的完整追溯链路。

   - *典型场景*："如果我修改这条接口需求，会影响哪些测试用例？""这条系统需求是从哪个用户需求派生的？"
   - *价值*：解决变更影响评估依赖人工、追溯链路维护困难的痛点，**加速变更评估**，减少因遗漏导致的返工。
4. **智能需求生成**：从上位文档自动拆解、派生下位需求，支持多形式输入转化。

   - *典型场景*："根据这份会议纪要，帮我生成用户需求""这条系统需求需要分解到软件层，请生成派生需求"。
   - *价值*：解决需求编写耗时、非结构化输入难以转化的痛点，**提升需求编写效率**，减少派生工作量。
5. **过程引导**：内嵌的"需求工程专家"，引导团队按最佳实践执行。

   - *典型场景*："当前阶段我应该做什么？有哪些输出物要交付？""这个项目的需求管理是否符合GJB5000A要求？"
   - *价值*：解决新手工程师不熟悉流程、方法论培训成本高的痛点，**提升流程合规性**，降低培训成本。

**【我们怎么做：实施路径】**

- **阶段一（2025.Q4）**：完成核心Agent开发，对接瑶光API（通过MCP），实现查询与审查功能的内测。
- **阶段二（2026.Q1）**：在1-2个典型项目中试运行，通过RLHF（人类反馈强化学习）优化回答准确率，打磨"变更分析"的准确度。
- **阶段三（2026.Q2）**：**能力平台化输出**——将摇光小智的实现路径（Agent框架+知识库+MCP连接器）封装为可复制的能力模块。不仅作为瑶光的增值服务销售，更重要的是**将这套能力输出到客户现有系统中**：客户无需替换其Doors、JIRA等现有工具，我们通过连接器读取其系统数据，在其环境中部署一个专属的"AI需求助手"，实现同等的智能问答、质量审查、变更分析能力。

**【商业拓展思考：更多可能性】**

如果摇光小智在瑶光系统中跑通了，它的核心价值不仅是"瑶光的插件"，而是**"企业级知识伴侣"**：

- **拓展方向：客户系统的"外挂大脑"**
  - 很多客户已有成熟（甚至老旧）的需求系统，他们很难推翻重来。
  - **新模式**：我们将摇光小智的AI能力（Agent + 知识库）独立封装，作为"外挂"部署在客户侧，通过连接器读取他们现有系统的数据。
  - **价值**：不替换客户系统，也能让客户享受到AI带来的效率提升，极大降低了进场门槛。

### 2.2 玑衡自绘：从"AI辅助建模"到"自动化工程服务"

**【我们要做什么：具象产品能力】**

玑衡自绘是集成在**玉衡建模工具**中的AI引擎，致力于降低MBSE建模门槛，让建模从"人工绘制"走向"AI辅助生成"。我们将重点实现两大核心能力：

1. **文本转模型（Text-to-Model）**：输入自然语言描述，自动生成符合SysML规范的图形模型。

   - *典型场景*："建立一个包含电源、控制器和电机的驱动系统""根据这段需求描述，生成对应的用例图"。
   - *价值*：解决建模门槛高、人工绘制耗时的痛点，**大幅降低建模入门门槛**，让非专业人员也能快速产出模型初稿。
2. **模型转文档**：从模型一键生成工程文档（.xml），实现模型资产的标准化输出。

   - *典型场景*："把这个系统架构模型导出为标准XMI格式""生成这个模块的设计说明文档"。
   - *价值*：解决模型与文档脱节、人工撰写文档重复劳动的痛点，**实现模型-文档一体化**，提升交付物产出效率。

**【我们怎么做：实施路径】**

- **阶段一（2025.Q4）**：完成"文本生成模型"核心能力，用户输入一段话，AI生成XMI文件供导入玉衡。解决"从0到1"的快速构建问题。
- **阶段二（2026.Q2）**：实现"Copilot交互"模式，在画布上实时通过对话修改模型（增删改查），解决"精确调整"问题。

**【商业拓展思考：更多可能性】**

玑衡自绘的核心价值在于**将非结构化信息（语言/文档）转化为结构化工程数据（模型）**。这种能力一旦成熟，真正的商业拓展方向是：

- **拓展方向：AI建模能力的独立输出**
  - 很多客户已有成熟的建模工具（如Rhapsody、MagicDraw等），短期内不会切换到玉衡。但他们同样面临建模门槛高、效率低的痛点。
  - **新模式**：将玑衡自绘的AI建模能力封装为**独立的能力模块**，可以"嫁接"到客户现有的建模环境中。客户无需更换工具，也能享受AI辅助建模的效率提升。
  - **价值**：从"卖工具"转变为"卖能力"，扩大商业边界；同时降低客户进场门槛，先用AI能力"绑定"客户，再逐步引导其迁移到我们的工具生态。

### 2.3 辰极智脑赋能项目：从"卖产品"到"输出能力"

**【战略定位】**

前两个方向（摇光小智、玑衡自绘）是"做工具"——把AI能力嵌入现有产品。而**辰极智脑赋能项目**是"输出能力"——将辰极智脑中台的核心能力作为AI赋能解决方案输出给行业客户，帮助他们建设智能化平台。

简单说：**客户有建设AI平台的需求，我们用辰极智脑中台帮他们搭建，收项目服务费。**

**【典型案例：山东电工变压器设计仿真试验一体化平台】**

山东电工正在规划建设"变压器设计仿真试验一体化平台"，面临以下典型痛点：

| 痛点         | 具体表现                                            |
| ------------ | --------------------------------------------------- |
| 流程割裂     | 设计、仿真、试验环节数据不互通，参数手工转录耗时2天 |
| 仿真门槛高   | 报错信息晦涩难懂，青年工程师频繁求助专家            |
| 历史数据沉睡 | 相似问题重复踩坑，找不到历史资料                    |
| 经验流失     | 专家退休=知识流失，新人培养周期长达5年              |

**辰极智脑如何赋能**：

## 三、核心竞争壁垒

### 3.1 壁垒体系结构

公司在AI+MBSE领域构建三层**并行积累**的竞争壁垒：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           三层壁垒并行积累                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────┐  ┌─────────────────────┐  ┌─────────────────────┐  │
│  │    第一层壁垒        │  │    第二层壁垒        │  │    第三层壁垒        │  │
│  │   行业知识资产       │  │   客户项目数据       │  │   场景深度耦合       │  │
│  │  积累周期：2-3年     │  │  积累周期：5-10年    │  │  积累周期：3-5年     │  │
│  └─────────────────────┘  └─────────────────────┘  └─────────────────────┘  │
│           │                         │                         │            │
│           └─────────────────────────┴─────────────────────────┘            │
│                                     │                                       │
│                                     ▼                                       │
│                          【相互增强、协同提升】                              │
│           知识沉淀 ←→ 项目积累 ←→ 场景绑定 ←→ 形成综合竞争壁垒              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**关键认知**：三层壁垒是**并行积累**而非串行依赖。在积累行业知识资产的同时，通过客户项目积累项目数据，通过产品落地实现场景耦合。三者相互增强：

- 行业知识让产品更专业 → 更容易获得客户项目
- 客户项目产生使用数据 → 反哺知识库，AI更懂客户
- 场景深度耦合 → 客户使用更频繁，数据积累更快

### 3.2 第一层壁垒：行业知识资产

| 知识资产               | 内容                               | 复制难度             |
| ---------------------- | ---------------------------------- | -------------------- |
| **MBSE方法论**   | 公司多年积累的系统工程建模最佳实践 | 中等，需长期项目积累 |
| **行业合规规范** | GJB、航空航天等行业标准与合规要求  | 高，需深厚行业背景   |
| **SysML知识库**  | 规范解读、操作指南、常见问题解答   | 中等，需专业团队建设 |

**壁垒价值**：公司深耕MBSE领域多年，积累的行业知识已数字化沉淀在知识库中。竞争对手需要2-3年才能建立同等水平的知识资产。

### 3.3 第二层壁垒：客户项目数据

| 数据资产                       | 内容                           | 复制难度                  |
| ------------------------------ | ------------------------------ | ------------------------- |
| **历史项目模型**         | 多年交付的MBSE项目模型资产     | 极高，竞争对手无法获取    |
| **知识图谱（GraphRAG）** | 模型元素间的关联关系与语义网络 | 极高，需数据+技术双重积累 |
| **客户使用数据**         | 客户在系统中的操作、问答、偏好 | 极高，仅本公司独有        |

**壁垒价值**：客户使用产品的过程即是数据积累的过程。使用时间越长，知识图谱越丰富，AI越能理解客户的项目上下文。这种积累形成"**数据飞轮**"效应：

```
客户使用产品 → 产生数据 → 丰富知识库 → AI更理解客户 → 客户更依赖 → 更多使用
      ↑                                                              │
      └──────────────────────────────────────────────────────────────┘
```

### 3.4 第三层壁垒：场景深度耦合

| 耦合维度              | 内容                             | 复制难度             |
| --------------------- | -------------------------------- | -------------------- |
| **MCP协议集成** | 与瑶光/天玑/玉衡的实时上下文感知 | 极高，需产品团队配合 |
| **工作流嵌入**  | AI嵌入客户日常工作流程           | 高，需深度定制       |
| **私有化部署**  | 满足军工内网隔离要求             | 高，需完整解决方案   |

**壁垒价值**：公司AI与自研MBSE软件深度集成，形成"**场景锁定**"效应。竞争对手的AI产品难以获得同等深度的场景接入。

### 3.5 壁垒的商业意义

| 壁垒层级     | 商业效果                                 |
| ------------ | ---------------------------------------- |
| 行业知识资产 | 产品专业性强，客户初次选择时的差异化优势 |
| 客户项目数据 | 使用越久越懂客户，形成高切换成本         |
| 场景深度耦合 | 产品与客户工作流程深度绑定，难以替换     |

## 四、商业模式与收入规划

### 4.1 收入来源结构

| 收入类型              | 产品/服务               | 具体方式                                 |
| --------------------- | ----------------------- | ---------------------------------------- |
| **增值服务**    | 摇光小智+玑衡自绘       | 捆绑现有软件销售                         |
| **插件工具**    | 摇光小智/玑衡自绘独立版 | 作为独立插件单独销售，支持嵌入第三方工具 |
| **软件订阅**    | Create平台              | 年度订阅制                               |
| **项目交付**    | AI赋能项目              | 按项目收费                               |
| **License买断** | 私有化部署              | 一次性买断                               |
| **生态抽成**    | 智能体/模板市场         | 平台抽成                                 |

### 4.2 增值服务模式

**商业逻辑**：摇光小智、玑衡自绘作为现有软件（瑶光、天玑/玉衡）的AI增值包销售，提升客单价。

**销售路径**：

1. 新客户购买瑶光/天玑时，默认包含30天AI增值包试用
2. 老客户升级时，推荐AI增值包
3. 试用期结束前，销售跟进转化

**定价策略**：

- **摇光小智**：根据团队规模分档，按年收费
- **玑衡自绘**：根据建模量分档，按年收费

### 4.3 软件订阅模式

**商业逻辑**：Create平台采用SaaS订阅模式，按年收费，形成稳定现金流。

**版本分层**：

| 版本                 | 目标客户           | 定价策略                 |
| -------------------- | ------------------ | ------------------------ |
| **社区版**     | 学生、爱好者       | 免费（用户积累为主）     |
| **学习版**     | 专业学习者、小团队 | 低价订阅，按用户数分档   |
| **企业版**     | 中小企业           | 年度订阅，按团队规模分档 |
| **大型企业版** | 大型企业           | 年度订阅，按定制需求报价 |

### 4.4 项目交付模式

**商业逻辑**：AI辅助使建模效率大幅提升，同等团队可承接更多项目，项目利润提升。

**价值传导**：

- 传统模式：建一个中等系统需数周，多人配合
- AI辅助后：周期大幅缩短，人员精简
- 效率提升 → 承接能力提升 → 项目收入增长
- 人力成本降低 → 项目利润提升

**定价策略**：按项目规模和复杂度报价，维持价值定价（而非成本定价）

### 4.5 私有化部署模式

**商业逻辑**：军工、航空航天等涉密行业必须私有化部署，形成高单价收入。

**交付内容**：

- 完整系统私有化部署
- 数据完全在客户内网
- 年度技术支持与升级

**定价策略**：一次性买断 + 年度维护费（根据部署规模和定制需求报价）

### 4.6 生态抽成模式（远期）

**商业逻辑**：开放平台API，允许第三方开发智能体和模板，平台抽成。

**实施路径**：

1. 27年开放API接口
2. 建立智能体市场和模板市场
3. 制定开发者激励政策
4. 形成生态飞轮效应

## 七、总结

### 7.1 战略核心回顾

- **一个中台**：辰极智脑中台作为能力底座。
- **三层壁垒**：行业知识、客户数据、场景耦合。
- **四大方向**：
  - 摇光小智（做深）：嵌入瑶光，深度绑定需求场景
  - 玑衡自绘（做快）：嵌入玉衡，提升建模效率
  - Create平台（做广）：独立平台，打开新市场、建立生态
  - 辰极智脑赋能项目（做项目）：输出中台能力，帮客户搭建AI平台

### 7.2 核心竞争力

| 竞争力维度         | 具体内容                           |
| ------------------ | ---------------------------------- |
| **行业知识** | 深耕MBSE领域多年，沉淀行业知识资产 |
| **数据积累** | 客户使用形成数据飞轮，越用越强     |
| **场景耦合** | 与自研软件深度集成，形成场景锁定   |

### 7.3 商业目标

| 时间节点             | 收入目标          | 核心动作                 |
| -------------------- | ----------------- | ------------------------ |
| 2026年               | 1,300万           | 四大方向商业化启动       |
| 2027年               | 3,700万           | 规模化销售，生态初步形成 |
| **累计27年底** | **5,000万** | —                       |

### 7.4 核心价值主张

> **用AI重塑系统工程，让机器承担繁琐，让人专注创新。** 辰极AI不是通用AI工具，而是深度理解MBSE行业的AI工程能力平台。产品越用越懂客户的项目，形成"用得越久、越离不开"的正向循环。
