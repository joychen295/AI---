## 一、产品定位与概述

辰极智脑AI中台是公司AI战略的**核心基础设施**，定位为"公司级AI服务中台"，承担着统一AI能力供给、知识资产沉淀、智能服务编排三大使命。

### 1.1 产品背景

当前MBSE（模型驱动系统工程）行业正处于关键转型期。传统建模工具面临三大核心痛点：**学习成本高、建模效率低、协作能力弱**。据统计，工程师掌握专业MBSE工具平均需要6-12个月，单个中等复杂度系统建模耗时2-4周。

随着大语言模型技术的成熟，AI与工程软件的深度融合已成为行业共识。辰极智脑AI中台正是在此背景下，为公司全产品线提供统一的AI能力底座。

### 1.2 战略价值

| 价值维度 | 具体体现 |
|----------|----------|
| **技术领先** | 打造业内领先的AI+MBSE解决方案，建模效率提升10倍 |
| **商业突破** | 从软件销售向解决方案交付转型，实现"卖结果"而非"卖工具" |
| **生态构建** | 建立MBSE行业的开放平台，培育开发者社区 |

## 二、核心功能模块

### 2.1 企业知识中枢

#### 功能要点

**知识库体系建设**：构建覆盖MBSE领域的多层次知识库体系，包括：

- **本体论知识**：SysML规范、MBSE理论基础
- **方法论知识**：公司内部建模方法论、最佳实践
- **案例库**：历史项目典型模型，作为参考模板
- **外部知识库**：合作伙伴提供的MBSE行业知识

**知识检索引擎**：提供检索型与对话型两种交互模式的智能检索服务，支持本地、全局、混合、朴素四种检索策略，满足不同场景下的知识获取需求。

**知识库配置与管理**：支持本地部署与云端托管两种知识引擎模式，具备文档自动解析、向量化索引、知识图谱构建等完整的知识生产能力。

#### 解决的行业问题

| 行业痛点 | 解决方案 | 价值体现 |
|----------|----------|----------|
| MBSE领域知识分散，工程师培养周期长（平均6-12个月） | 构建覆盖率80%以上的行业知识库，提供即时检索与智能问答 | 新员工培养周期缩短50%以上 |
| 历史项目经验难以复用，相似系统重复建模 | 建立模型资产知识图谱，支持相似模型检索与参考 | 建模效率提升显著 |
| 规范标准分散，合规性校验依赖人工经验 | 沉淀行业规范至知识库，支持自动化规范查询 | 降低合规风险 |

---

### 2.2 文档资产管理

#### 功能要点

**多格式文档统一管理**：支持PDF、Word、Excel、PowerPoint、Markdown、HTML等14种文档格式及8种图片格式的上传、存储与管理。

**自动化处理流水线**：文档上传后自动触发解析、分块、向量化、索引构建的完整处理流程，实现非结构化数据的结构化转换。

**精细化分类体系**：支持文档分类、标签管理、权限控制，实现企业知识资产的规范化组织与个性化检索。

#### 解决的行业问题

| 行业痛点 | 解决方案 | 价值体现 |
|----------|----------|----------|
| 军工、航空航天企业技术文档数量庞大、格式多样，难以有效利用 | 提供统一存储与自动索引，支持模糊搜索与精准筛选 | 文档查找效率提升 |
| 项目交付后的技术沉淀分散，缺乏系统性管理 | 建立企业级文档中心，实现知识资产规范化沉淀 | 知识资产保值增值 |

---

### 2.3 智能检索与问答

#### 功能要点

**双模式交互**：

- **检索模式**：单次精准问答，强调答案的精准度与来源的可追溯性
- **对话模式**：多轮对话探索，支持上下文记忆，适合追问与探索

**多策略检索**：

| 模式 | 检索策略 | 适用场景 | 回答特点 |
|------|----------|----------|----------|
| **本地模式** | 仅基于文档块的相似度检索 | 精确查找某个文档中的具体内容 | 回答来源明确集中，速度快 |
| **全局模式** | 基于知识图谱的关系推理 | 跨多个文档的关联分析、总结归纳 | 回答综合性强，可能整合多个文档 |
| **混合模式** | 结合本地检索 + 全局推理 | 大多数通用问答场景（推荐默认） | 兼顾精准与全面 |
| **朴素模式** | 基础相似度检索，无知识图谱 | 知识图谱未构建或简单问答 | 回答直接，依赖原始文档块 |

**结果可追溯**：每次检索结果均提供来源引用、置信度评估、性能指标，确保答案的可验证性与透明度。

#### 解决的行业问题

| 行业痛点 | 解决方案 | 价值体现 |
|----------|----------|----------|
| 大型系统工程项目文档量大，关键信息查找耗时 | 智能语义检索替代关键词匹配，精准定位所需信息 | 检索效率提升 |
| MBSE规范理解门槛高，工程师需反复查阅手册 | 对话式问答即时获取专业解释与操作指导 | 学习成本降低 |
| 军工行业对信息准确性要求极高 | 答案附带来源引用与置信度，支持人工校验 | 信息可靠性保障 |

### 2.4 服务治理与监控（辰极脉络）

#### 功能要点

**服务自动发现与注册**：服务实例启动时自动注册，宕机时自动摘除，实现微服务的动态管理。

**服务拓扑可视化**：以网络拓扑图展示服务间调用关系：

- 节点颜色反映健康度（绿色=健康，红色=故障，黄色=警告）
- 连线粗细反映流量大小
- 点击节点可查看详情或高亮上下游关联

**智能诊断与告警**：

- 实时监听服务事件（新实例上线、心跳超时等）
- 核心服务下线时自动触发告警
- 记录完整的操作审计日志

#### 解决的行业问题

| 行业痛点 | 解决方案 | 价值体现 |
|----------|----------|----------|
| AI中台涉及多个子服务，运维复杂度高 | 统一服务治理平台，一站式监控与管理 | 运维效率提升 |
| 服务故障传递路径难以追溯 | 依赖关系可视化，快速定位故障源 | 故障恢复时间缩短 |
| 系统稳定性要求高（可用性≥99%） | 自动健康检测与告警机制 | 系统可靠性保障 |

| 角色 | 权限范围 |

#### 解决的行业问题

|----------|----------|----------|
| 军工企业对数据隔离与访问控制要求严格 | 租户级数据隔离，细粒度权限控制 | 合规性保障 |
| 大型企业多系统并行，账号管理复杂 | 对接统一认证中心，一套账号访问多系统 | 管理效率提升 |
| 系统运行状态不透明，问题发现滞后 | 实时仪表盘监控，主动发现潜在风险 | 运维前置化 |

## 三、技术架构

辰极智脑AI平台采用**微服务架构**，分为三层：

```
┌─────────────────────────────────────────────────────────────────┐
│                      业务应用层 (Application Layer)                │
├─────────────────────────────────────────────────────────────────┤
│  瑶光需求管理系统  │  天玑/玉衡建模工具  │  MBSE 2.0软件         │
└─────────────────────────────────────────────────────────────────┘
                                │
                       API / MCP 调用
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                   AI服务中台层 (AI Service Platform)               │
├─────────────────────────────────────────────────────────────────┤
│  服务管理：服务注册与发现、API网关、负载均衡、认证授权(SSO)          │
│  知识库管理：本地知识库(LightRAG)、外部知识库、模型知识库(GraphRAG)    │
│  AI工作流引擎：LangGraph编排、多智能体协作、记忆管理、上下文工程      │
│  大模型接入：DeepSeek、Kimi、GLM、自定义模型(可扩展)                │
└─────────────────────────────────────────────────────────────────┘
                                │
                       数据存储与计算
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                基础设施层 (Infrastructure Layer)                  │
├─────────────────────────────────────────────────────────────────┤
│  MySQL(关系数据) │ Neo4j(知识图谱) │ Qdrant(向量数据) │ Redis(缓存) │
│  Docker(容器化)  │ K8s(编排)      │ GPU服务器(8-16卡H20)           │
└─────────────────────────────────────────────────────────────────┘
```

### 核心设计理念：Human-in-the-Loop（人在回路）

系统采用"**AI辅助决策、人工最终确认**"的设计理念，确保在AI自动化生成、建模、推理等关键环节保留人工确认：

- **AI生成 + 人工审核**：AI生成结果需经工程师审核确认后方可正式写入
- **关键决策人工介入**：系统架构调整、关键需求变更等决策点，AI提供建议，人工最终决策
- **可解释性增强**：每个建议附带推理依据，支持引用溯源
- **渐进式自动化**：提供保守/标准/激进三种自动化级别设置
